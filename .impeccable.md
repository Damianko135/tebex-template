## Design Context

### Users

Two distinct audiences on one codebase:

- **Storefront customers** — players browsing and buying in-game items, ranks, and subscriptions
  for a game server. Game-agnostic: this template must work equally well for a Minecraft server,
  a FiveM server, a Rust server, an ARK server, etc., without feeling tailored to any one
  community. They're here to complete a purchase with confidence, not to be immersed in a themed
  experience.
- **Store admins** (`/admin`) — the server owner/operator managing catalog visibility, baskets,
  tiers, and the storefront's theme. Internal tool, not a marketing surface.

### Brand Personality

**Professional, trustworthy, storefront-focused.** Reads as a polished e-commerce platform first;
gaming is the *context*, not the *aesthetic*. Should inspire purchase confidence and feel
approachable to both server owners and players — not a "gamer" skin.

### Aesthetic Direction

**Editorial/magazine, light mode.** Confident whitespace, clear typographic hierarchy,
product-first layouts — closer to a modern SaaS or editorial site than a game storefront.

- **Reference**: Hypixel for content organization/storefront hierarchy only, never for visual
  styling. Broader inspiration: modern editorial and SaaS sites — clean typography, generous
  whitespace, clear hierarchy, product-first layouts.
- **Explicitly avoid**: dark themes as default, neon glows, oversized gradients, sci-fi styling,
  glassmorphism, heavy visual effects — the standard "gamer store" tropes.
- **Theme**: light is the shipped default. Both light and dark token sets stay fully implemented
  (existing infrastructure, `lib/ui/tokens.ts`) — dark isn't the target aesthetic, but it must
  remain coherent since a merchant can still switch to it.
- **Color**: this is a reusable template — the default palette is intentionally neutral, chosen to
  suit an editorial/light aesthetic and work as a sensible starting point, not to express a strong
  brand identity or to match Tebex's own branding. Flexibility over identity: merchants reskin via
  Admin → Theme (color tokens only, not typography/layout/motion).

### Design Principles

1. **Product-first, not gamer-first.** Typography, layout, and color should read as e-commerce/
   editorial. Any "gaming" signal should come from the actual product content (item names, server
   stats), never from the chrome around it.
2. **Game-agnostic neutrality.** Nothing in the default template should assume or imply a specific
   game, art style, or community.
3. **Confidence over decoration.** Every visual choice should make a purchase feel safe and clear.
   No effects for their own sake.
4. **Merchants own color; the template owns everything else.** Typography scale, spacing rhythm,
   motion, and component structure are the template's job and should stay stable across whatever
   palette a merchant applies via the theme editor.
5. **Accessibility is table stakes, not a feature.** WCAG-conformant contrast, full keyboard
   navigation, visible focus states, and respect for `prefers-reduced-motion` throughout — no
   separate "accessible mode."
